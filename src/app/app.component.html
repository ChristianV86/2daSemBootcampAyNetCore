7. Creación de módulos

Es preferible separar en módulos la aplicación. Los módulos permiten dividir el proyecto en secciones. Estas secciones deben ser de aspectos importantes del sistema, como por ejemplo, un módulo para empleados, otro para clientes, otro para productos, etc.
El principal objetivo es encapsular varios componentes y ayudar tambien con el lazy loading o carga peresoza; esto permite que solo se cargue la información que está dentro del módulo.

Pasos para creación de módulos

Vamos a la consola y ejecutamos el siguiente comando: ng generate module nombre_modulo, la forma corta sería: ng g m nombre_modulo. Se acostumbra que el nombre del módulo sea el de un directorio raiz ya creado para que no se cree a nivel del directorio principal del proyecto.
